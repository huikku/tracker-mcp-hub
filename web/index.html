tracker-mcp-hub is a tracker-agnostic production-data layer. Each tracker MCP emits the same normalized snapshot, and the hub turns that one contract into things no single tracker can do — verify, migrate, audit, snapshot and roll up across ShotGrid, ftrack, Kitsu and AYON interchangeably.

shotgrid-mcp ftrack-mcp kitsu-mcp ayon-mcp

Capabilities

Five tools over one normalized contract
verify
Diff two project snapshots → a report: count deltas, missing/extra entities, casting & per-task status mismatches, coverage — and a match / discrepancies verdict.
migrate
Orchestrated copy: read → write → verify. Every run ends with proof. Preflight before, an automatic verify report after.
audit
Verify one source-of-truth against many mirror copies → a drift report. Ideal on a schedule: are my mirrors still faithful?
snapshot
Wrap a project into a portable, versioned archive (the normalized model + a media manifest) — restorable to any tracker.
rollup
Aggregate many projects into totals, a status distribution and coverage — a “state of all productions” dashboard, across trackers.
